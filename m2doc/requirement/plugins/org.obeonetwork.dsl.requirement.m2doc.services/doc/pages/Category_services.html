Services and attributes for the Category concept

Category instances are used to structure requirement repositories into sections and sub sections.

Attributes

Name Returned type M2Doc Syntaxe Description Example
technicalid String {m:category.technicalId} Technical identifier of the category The category's technical ID is {m:cat.technicalId}
name String {m:category.name} The name of the category
ownedCategories List of Categories {m:cat.ownedCategories} List of containedcategories (directly contained categories)
ownedRequirements List of Requirements {m:cat.ownedRequirements} List of directly contained requirements The contained requirements : {m:for req | cat.ownedRequirements} {m:req.name} {m:endfor}
requirements List of Requirements {m:cat.requirements} List of contained requirements
subCategories List of Categories {m:cat.subCategories} List of contained categories
id String {m:cat.id} category's identifier
repository Repository {m:cat.repository} repository where the category is contained
parentCategory Category {m:cat.parentCategory} parent category containing this one.
referencedObject List of Object {m:cat.referencedEObject} list of referenced objects

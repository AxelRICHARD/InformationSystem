Services and attributes for the Repository concept

Repository instances are the higest level container of requirements. They contain requirements but also requirement categories.

Attributes

Name Returned type M2Doc Syntaxe Description Example
technicalid String {m:repo.technicalId} Technical identifier of the repository Technical identifier of the repository The repository's technical ID is {m:repo.technicalId}
name String {m:repo.name} The name of the repository The repository's name is {m:repo.name}
ownedCategories List of Category {m:repo.ownedCategories} the list of categories referenced by this repository
ownedRequirements List of Requirements {m:repo.ownedRequirements} the list of requirements contained in the repository
mainCategories List of Categories {m:repo.mainCategories} list of root categories of the repository The requirement categories in this repository are {m:repo.mainCategories->sep(',')}
referencedObject List of Object {m:repo.referencedObject} list of object concerned by this requirement category

Services and attributes for the Requirement concept

Requirement instances are used to represent unit of system needs. They can be functional or technical.

Attributes

Name Returned type M2Doc Syntaxe Description Example
technicalid String {m:req.technicalId} Technical identifier of the requirement The requirement's technical ID is {m:req.technicalId}
name String {m:req.name} The name of the requirement The requirement's name is {m:req.technicalId}
id String {m:req.id} The business identifier of the requirement The requirement's identifier is {m:req.technicalId}
version String {m:req.version} The version of the requirement The requirement's version is {m:req.version}
referencedObject List of Objects {m:req.referencedObject} The list of object the requirement holds on: {m:for object | req.referencedObjects} . label : {m:object} {m:endfor} The list of object's this requirement holds on :
statement String {m:req.statement} The statement is the text that expresses the requirement
rationale String {m:req.rationale} The requirement's rationale.
acceptanceCriteria String {m:req.acceptanceCriteria} The acceptance criteria provides a criteria that must hold for the client to accept the specified system.
type RequirementType {m:req.type} The requirement type can be functionnal or technical
category Category {m:req.category} The category that contains the requirement
subtype String {m:req.subType} Application dependent requirement type.
status String {m:req.status} Application dependent status that indicate where the requirement lies in the requirement's life cycle.
createdOn Date {m:req.createdOn.toString()} Date where the requirement has been created.
modifiedOn Date {m:req.modifiedOn.toString()} Date where the requirement has been last modified.

Services

Name Returned type M2Doc Syntaxe Description Example
relatedRequirements List of Requirements {m:obj.relatedRequirements()} Returns the list of requirements that have the object 'obj' in it's referencedObject's field
relatedRequirementsWithSubtype List of Requirements with given subtype {m:obj.relatedRequirementsWithSubtype()} Returns the list of requirements that have the object 'obj' in it's referencedObject's field and are of the given subtype (which can be empty)

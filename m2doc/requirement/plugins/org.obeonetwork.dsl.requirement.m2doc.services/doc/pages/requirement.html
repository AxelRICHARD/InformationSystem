requirement

Overview

This documentation describes services and attributes available on requirement models in M2Doc generation templates.

Each concept (Repository, Category, Requirement, etc.) is described in a page of it's own. Services and attributes are described using

  • a name,
  • the syntax to be used in M2Doc generation templates,
  • a description of the attribute or service

For some attributes and services, a usage example is provided.

Documentation of M2Doc Services for requirement

Each concept of requirement is documented in it's own page :
  • M2Doc Services and Attributes for Repository
  • M2Doc Services and Attributes for Category
  • M2Doc Services and Attributes for Requirement
  • M2Doc Services and Attributes for EObject

Viewpoints

There are two representations available on requirements' Repository instances :
  • Requirements Table : a table that allows to edit the requirements.
  • Requirements Traceability Matrix : a table that traces the relation between objects on which the requirements hold and the requirements themselves.
